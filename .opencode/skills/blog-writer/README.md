# 博客写作助手 Skill

## 简介

这是一个帮助撰写高质量博客文章的Claude Code Skill。提供多种文章类型模板，支持模仿不同写作风格，遵循专业写作规范。

## 功能特性

- **三种写作模式**: 从头创作 / 提供素材创作 / 扩展已有内容
- **多种文章类型**: 技术分享、产品分析、观点评论、学习笔记、工具推荐、经验分享、成长故事
- **风格模板参考**: 专业严谨、轻松幽默、简洁明了、故事叙述、朴实稳重
- **字数灵活控制**: 支持500-5000字不同篇幅
- **素材输入功能**: 支持在对话中直接提供经历、感想、笔记等素材 ✨ 新增
- **智能标题提炼**: 未指定标题时，根据内容自动提炼吸引人的标题
- **自动保存归档**: 文章完成后自动保存到 archives 目录
- **结构化写作流程**: 从需求确认到最终输出的完整流程
- **质量保证**: 遵循专业写作规范，确保内容质量

## 使用方法

### 基本用法

#### 模式一：从头创作

```bash
# 写作技术分享文章（默认2000字）
/blog-writer 如何使用Spring Boot构建RESTful API

# 指定文章类型
/blog-writer ChatGPT对开发工作的影响 --type=观点评论

# 指定写作风格
/blog-writer Docker容器化实践指南 --type=技术分享 --style=轻松幽默

# 指定字数（简短版）
/blog-writer Git常用命令 --type=工具推荐 --words=500

# 指定字数（深度版）
/blog-writer 微服务架构设计 --type=技术分享 --words=5000

# 完整示例
/blog-writer 我从大厂面试中学到的东西 --type=经验分享 --style=故事叙述 --words=2000
```

#### 模式三：提供素材创作

```bash
# 交互式创作，系统会询问是否提供素材
/blog-writer 我最近经历了一次技术升级，想写一篇成长故事

# 系统会引导你：
# 1. 选择主题类别、文章类型、写作风格、目标字数
# 2. 询问是否要提供素材（经历、感想、笔记等）
# 3. 如果选择"是"，你可以：
#    a) 直接粘贴素材
#    b) 说明从 user-input 目录读取文件
# 4. 系统会将你的素材融入文章创作
```

**可提供的素材类型**：
- **经历内容**：工作项目、学习过程、生活事件
- **感想思考**：对某事的看法、感悟、反思
- **笔记草稿**：零散的笔记、未完成的想法
- **具体信息**：时间、地点、人物、数据等
- **示例案例**：具体的例子、案例、场景
- **特殊要求**：文章中需要包含的内容、角度、重点

**提供素材的方式**：

**方式1：直接粘贴**
```
系统询问：是否要提供素材？
用户选择：是
用户粘贴：
"我最近做了一个微服务升级项目...
主要经历了这些挑战...
最终性能提升了3倍..."
用户说：素材提供完毕
```

**方式2：从 user-input 目录读取**
```
系统询问：是否要提供素材？
用户选择：是
用户说明：请从 user-input 目录读取 微服务升级经历.md
系统读取文件并分析素材
```

#### 模式二：扩展已有内容

```bash
# 方式1：提供已有内容文件
/blog-writer --input=./my-ideas.md --type=技术分享 --style=简洁明了

# 方式2：提供内容文件并指定字数
/blog-writer --input=./draft.md --type=技术分享 --words=2000

# 方式3：使用时说明有内容（随后粘贴）
/blog-writer 帮我扩展成技术博客
```

**可扩展的内容类型**：
- **想法碎片**: 零散的观点、笔记、待整理的想法
- **草稿**: 不完整的文章草稿
- **初稿**: 已写好但需要润色和扩展
- **大纲**: 文章结构框架
- **代码片段**: 需要围绕代码写教程

### 参数说明

| 参数 | 说明 | 可选值 | 默认值 |
|------|------|--------|--------|
| `--type` | 文章类型 | 技术分享、产品分析、观点评论、学习笔记、工具推荐、经验分享、成长故事 | 技术分享 |
| `--style` | 写作风格 | 专业严谨、轻松幽默、简洁明了、故事叙述、朴实稳重 | 专业严谨 |
| `--words` | 目标字数 | 500、1000、1500、2000、3000、5000 | 2000 |
| `--input` | 已有内容文件路径 | 任意文本文件路径 | 无 |

### 字数说明

| 字数 | 适用场景 | 内容深度 | 阅读时间 |
|------|----------|----------|----------|
| 500 | 快速阅读、要点速记 | 核心概念 + 基础用法 | 2-3分钟 |
| 1000 | 标准博客、日常分享 | 完整流程 + 简要案例 | 4-5分钟 |
| 1500 | 较详细版、进阶文章 | 较完整阐述 + 适度分析 | 6-8分钟 |
| 2000 | 深度文章、技术干货 | 全面阐述 + 详细代码 | 8-10分钟 |
| 3000 | 深度长文、案例分析 | 全面覆盖 + 案例丰富 | 12-15分钟 |
| 5000 | 系列文章、权威指南 | 源码分析 + 性能对比 | 20-25分钟 |

## 写作风格说明

### 专业严谨
- 学术化的表达方式
- 数据和事实支撑观点
- 逻辑严密，论证充分
- 适合技术深度文章

### 轻松幽默
- 生动有趣的表达
- 丰富的比喻和类比
- 语言活泼，轻松易读
- 适合入门教程和经验分享

### 简洁明了
- 短小精悍的内容
- 直击要点的阐述
- 快速阅读体验
- 适合快节奏读者

### 故事叙述
- 以故事为主线
- 循序渐进的展开
- 引人入胜的情节
- 适合经验分享和感悟文章

### 朴实稳重
- 语言朴实不浮夸
- 情绪稳定，平和客观
- 实事求是，不夸大
- 稳重踏实，给人可靠感
- 适合经验分享、技术总结、踩坑记录

### 扩展策略说明

| 已有内容类型 | 扩展策略 |
|-------------|----------|
| **想法碎片** | 提炼观点，按模板结构组织，补充案例和代码 |
| **草稿** | 保留核心内容，补全缺失章节，增强逻辑性 |
| **初稿** | 润色语言，调整结构，增加深度分析 |
| **大纲** | 按结构填充内容，添加代码示例和案例 |
| **代码片段** | 围绕代码写教程：原理→用法→实战→进阶 |

**扩展原则**：
- 保留精华：用户的核心观点和原创内容优先保留
- 补充结构：按模板添加缺失的章节（开头、结尾等）
- 深化内容：在用户观点基础上增加论证和案例
- 统一风格：让扩展部分与用户原有内容风格一致

## 素材输入功能

### 如何使用

使用 `/blog-writer [主题]` 调用 skill 后，系统会引导你完成以下步骤：

1. **选择主题类别**（运动健康、投资理财、生活杂谈、软件开发）
2. **选择文章类型**（技术分享、产品分析、观点评论、学习笔记、工具推荐、经验分享、成长故事）
3. **选择写作风格**（专业严谨、轻松幽默、简洁明了、故事叙述、朴实稳重）
4. **选择目标字数**（500/1000/1500/2000/3000/5000）
5. **询问是否提供素材** ✨

### 提供素材的选项

系统会询问：
```
是否要提供素材？（你的经历、感想、笔记、草稿、信息、示例、要求等）
- 是：我将提供素材
- 否：不需要，直接开始创作
```

### 可以提供什么素材

你提供的素材可以帮助AI写出更贴合你实际情况的文章：

- **经历内容**：工作项目、技术升级、学习过程、生活事件
- **感想思考**：对某事的看法、感悟、反思、总结
- **笔记草稿**：零散的笔记、未完成的想法、思维片段
- **具体信息**：时间、地点、人物、数据等细节
- **示例案例**：具体的例子、案例、场景
- **特殊要求**：文章中需要包含的内容、角度、重点
- **任何信息**：你认为对文章有帮助的内容

### 如何提供素材

**方式1：直接粘贴**
1. **选择"是：我将提供素材"**
2. **直接粘贴或输入**你的素材
3. **可以分多条发送**，不需要一次性全部提供
4. **发送完毕后**，告诉系统"素材提供完毕"或"完成了"

**方式2：从 user-input 目录读取**
1. **选择"是：我将提供素材"**
2. **说明要读取的文件**，例如：
   - "请从 user-input 目录读取 微服务升级经历.md"
   - "素材：user-input/我的学习笔记.txt"
3. **系统会自动读取文件**并分析内容
4. 确认素材后继续创作

### user-input 目录说明

`user-input/` 目录用于存放素材文件，方便重复使用：

**目录位置**：
```
.opencode/skills/blog-writer/user-input/
```

**使用场景**：
- 你有经常想要使用的素材
- 素材内容较长，不适合直接粘贴
- 希望保存素材以便将来使用

**文件格式**：
- 支持 `.md`、`.txt` 等文本格式
- 文件名可以任意命名，建议有意义
- 内容格式可以很随意，不要求完美

**使用示例**：
```
# 在 user-input 目录创建素材文件
user-input/我的技术升级经历.md
user-input/Vue学习笔记.txt
user-input/产品经理的思考.md

# 在写作时引用
"请从 user-input 目录读取 我的技术升级经历.md"
```
3. **可以分多条发送**，不需要一次性全部提供
4. **发送完毕后**，告诉系统"素材提供完毕"或"完成了"

### 素材如何被使用

AI 会：
- ✅ **分析素材内容**，提炼关键信息
- ✅ **保留核心观点**，确保你的原意不被改变
- ✅ **融入文章结构**，使内容更完整
- ✅ **适当润色组织**，提升表达质量
- ✅ **保持风格一致**，使文章更专业

## 文章保存

### 自动归档

文章完成后会自动保存到 `archives/` 目录，文件名格式：

```
[主题类别] | [文章标题].md
```

示例：
- `运动健康 | 极限健身：为久坐一族编写的健康手册.md`
- `软件开发 | 代码审查：从"找茬"到"互助"的心路历程.md`

### 智能标题提炼

如果你没有指定文章标题，AI 会：
1. 分析文章内容
2. 提炼核心观点
3. 生成3-5个标题选项
4. 推荐最合适的标题
5. 征求你的意见

标题标准：
- ✅ 吸引眼球，准确概括内容
- ✅ 包含关键词，便于搜索
- ✅ 长度适中（通常10-25字）
- ✅ 有吸引力但不标题党
